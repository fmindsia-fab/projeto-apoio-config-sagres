Configurações Gerais

Tela principal de parâmetros globais do sistema Squalus Gestão.

Print da Tela Salve o print como prints/cfg_gerais.png para exibir aqui.
Como acessar: Abra o módulo Gestão → clique em Configurações Gerais. Apenas usuários com permissão de administrador conseguem alterar esses campos.
Dados da Empresa
Campo Tipo Descrição / Orientação de Suporte
Nome da Empresa (p/Relatórios)
 Texto Nome exibido no cabeçalho de todos os relatórios impressos pelo sistema. Não interfere no cadastro fiscal nem no CNPJ da empresa — é apenas para fins de impressão.
Endereço - Linha 1
 Texto Primeira linha de endereço/contato impressa nos relatórios. Frequentemente utilizada para o telefone do estabelecimento ou o logradouro completo.
Endereço - Linha 2
 Texto Segunda linha de endereço impressa nos relatórios. Normalmente usada para Cidade – Estado ou complemento do endereço.
Diretório do Sistema Gestão (indicar caminho via rede)
 Caminho UNC Caminho de rede (UNC) para a pasta de instalação do Gestão no servidor. Obrigatório em ambientes multi-estação para que os PDVs localizem o banco de dados central. Deixar em branco quando a instalação for local (servidor e PDV na mesma máquina). Exemplo: \\SERVIDOR01\Squalus
Cadastro de Produtos
CampoPadrãoDescrição
Usar Filtro no campo "Descrição" p/o Cad. Produtos
 S-Sim Quando S-Sim, a pesquisa de produtos no cadastro funciona por filtro dinâmico: o sistema exibe os resultados enquanto o usuário digita, sem precisar pressionar Enter. Quando N-Não, é necessário pressionar Enter para acionar a busca.
Usar Campo "Cobrar Tx. Serviço" no Cadastro de Produtos
 N-Não Exibe o campo "Cobrar Taxa de Serviço" na ficha de cada produto, permitindo definir individualmente quais itens do cardápio incluem a taxa de serviço. Ativar somente se o restaurante cobra taxa por produto específico, e não de forma global.
Usar Campo "Balança Alternativa" p/os Produtos
 N-Não Habilita um campo no cadastro de produto para vincular uma balança alternativa (secundária). Útil em estabelecimentos que operam com mais de um tipo ou marca de balança integrada ao sistema.
Usar Agrupamento Interligado ao Grupos de Produtos
 N-Não Quando S-Sim, o agrupamento de produtos fica vinculado diretamente à estrutura de grupos cadastrada no sistema, mantendo hierarquia coerente entre grupo e agrupamento. Impacta a exibição em relatórios e na tela do PDV.
Utilizar "Combo" p/Produtos
 Não Ativa o módulo de montagem de combos. Permite criar produtos compostos por outros itens do cardápio com regras de preço e substituição. Requer configuração adicional no cadastro de produtos após habilitar.
Permitir Exclusão de "Outros Códigos" dos Produtos
 N-Não Quando S-Sim, o usuário pode excluir códigos de barras alternativos (EAN, balança ou PDV) cadastrados em um produto. Manter N-Não para evitar exclusão acidental de códigos vinculados a equipamentos de leitura.
Clientes e Entregas
CampoPadrãoDescrição
Utilizar Campo "Número" nos Endereços
 S-Sim Quando S-Sim, o campo de número do imóvel é exibido separadamente do logradouro no cadastro de clientes e na tela de entregas. Facilita a integração com sistemas de roteirização e mapa.
Usar Campo "Celular p/SMS" no Cad. de Clientes
 N-Não Adiciona um campo específico de número de celular no cadastro de clientes destinado ao envio de SMS (confirmação de pedido, promoções). Ativar somente se houver módulo de SMS contratado e configurado.
Utilizar Código de Clientes nas Reservas
 S-Sim Quando S-Sim, ao registrar uma reserva de mesa o sistema vincula o código do cliente cadastrado, permitindo consultar histórico de reservas por cliente e facilitar o atendimento na chegada.
Apenas Dígitos nos Telefones das Entregas
 S-Sim Quando S-Sim, o campo de telefone na tela de entregas aceita somente números, sem traços, parênteses ou espaços. Padroniza o formato e facilita integração com discagem automática ou WhatsApp.
Usar Campo "Tx. Entrega p/Repasse"
 N-Não Adiciona um campo de taxa de entrega individual por pedido, cujo valor é repassado ao entregador no fechamento do caixa. Ativar quando o estabelecimento opera com entregadores próprios e repassa a taxa de delivery a eles.
Validar Telefone Celular nas Entregas
 N-Não Quando S-Sim, o sistema valida o formato do número de celular antes de confirmar o pedido de entrega. Útil para evitar erros de digitação e garantir que o número seja válido para contato ou envio de SMS/WhatsApp.
Obrigar Bloqueio "Compras a Prazo" ou "Venda Antecipada" p/Clientes
 N-Não Quando S-Sim, o sistema exige que o campo "Bloqueado para Compras a Prazo" seja preenchido explicitamente no cadastro do cliente antes de salvar. Evita clientes com situação de crédito indefinida.
Valor Padrão p/Clientes Campo "Bloqueado p/Compras a Prazo"
 N-Não Define o valor padrão do campo "Bloqueado para Compras a Prazo" ao cadastrar um novo cliente. Quando S-Sim, todo cliente novo já é criado bloqueado para crédito — o operador precisa liberar manualmente. Quando N-Não, clientes novos chegam liberados por padrão.
Preços e Tabelas
CampoPadrãoDescrição
Escolher Empresa na Alteração de Preços de Produtos
 N-Não Em ambientes com mais de uma empresa cadastrada, quando S-Sim o sistema pergunta qual empresa terá os preços alterados antes de aplicar a mudança. Quando N-Não, a alteração afeta sempre a empresa logada.
Pode Alterar Preços c/Base Em Outra Tabela
 N-Não Quando S-Sim, permite usar os valores de uma tabela de preços alternativa como referência para atualizar os preços da tabela principal (ex.: reajustar tabela 1 com base nos preços da tabela 2 mais um percentual).
Pode Alterar Preços c/Base no Markup
 Apenas nas notas de compra Controla em quais momentos o sistema permite recalcular o preço de venda automaticamente com base no markup (margem) configurado por produto.
Sempre: permite em qualquer alteração de custo. Apenas nas notas de compra: recalcula somente ao lançar uma nota de entrada. Nunca: desativa o recálculo automático.
Financeiro e Contas
CampoPadrãoDescrição
Permitir Recebimento de Contas Assinadas no Gestão
 N-Não Quando S-Sim, habilita o recebimento, dentro do módulo Gestão, de contas que foram assinadas eletronicamente pelo cliente no ECF/PDV (ex.: comanda assinada). Quando N-Não, esse recebimento só pode ser feito diretamente no caixa do PDV.
Receber Conta Assinada de Quais Empresas no ECF
 1-Todas as Empresas Define de quais empresas o ECF/PDV pode receber contas assinadas. Utilizado em operações multi-empresa com caixas compartilhados entre filiais. Selecionar a empresa específica quando for necessário restringir o recebimento.
Permitir Alterar Valores das Formas de Pagamento dos Caixas no Gestão
 N-Não Quando S-Sim, o usuário do módulo Gestão pode editar os valores das formas de pagamento já informados no fechamento de caixa do PDV (ex.: corrigir valor de dinheiro vs. cartão após fechamento). Manter N-Não para preservar integridade dos fechamentos.
PDV e Operação
CampoPadrãoDescrição
Módulo de Biometria
 Não Ativa o suporte a leitor biométrico para autenticação de operadores no PDV (em substituição ou complemento ao cartão de código de barras). Requer hardware de biometria instalado e driver configurado no sistema operacional.
Ajustar Configurações de Internet ao Entrar no Sistema
 N-Não Quando S-Sim, o sistema redefine configurações de proxy e internet ao iniciar. Usar somente se o cliente relatar conflitos de rede no momento da abertura do sistema — em situações normais deve permanecer N-Não.
Registrar Itens no Cancelamento Total nas Ocorrências do Caixa
 N-Não Quando S-Sim, todos os itens de um pedido cancelado integralmente são detalhados no log de ocorrências do caixa, facilitando auditoria e controle de cancelamentos por operador.
Checar Valores de Pré-Conta (sempre deixar como S-Sim)
 S-Sim Manter sempre como S-Sim. O sistema valida os valores da pré-conta (subtotal do pedido aberto) antes de efetuar o fechamento, garantindo que o valor cobrado corresponde ao que foi consumido. Alterar para N-Não pode causar divergências de valor entre a pré-conta e o documento fiscal emitido.
Ação a Ser Tomada Quando Operador Abrir Mais de Um PDV
 3-Não Permitir Controla o comportamento quando o mesmo operador tenta abrir um segundo terminal PDV simultaneamente. 1-Avisar: exibe alerta mas permite abrir. 2-Fechar o Anterior: encerra automaticamente a sessão no primeiro PDV. 3-Não Permitir: bloqueia a abertura do segundo PDV. Recomendado: opção 3 para evitar conflitos de caixa.
Criptografar Senha do Operador no Cartão Cód. Barras
 S-Sim Quando S-Sim, a senha do operador é gravada de forma criptografada no cartão de código de barras, impedindo leitura direta da senha em caso de clonagem do cartão. Manter sempre S-Sim por segurança.
Mensagem de Backup Preventivo
 S-Sim Quando S-Sim, o sistema exibe um lembrete de backup ao ser aberto. Manter S-Sim para reforçar a rotina de backup do cliente e evitar perda de dados em caso de falha de hardware.
Tecla de Atalho (Shift+F10) p/Relatórios
 Não Quando ativado, o atalho de teclado Shift+F10 abre a tela de relatórios de qualquer ponto do sistema, sem precisar navegar pelos menus. Útil para estabelecimentos que consultam relatórios com frequência durante a operação.

Configurações Gerais 2

Layout do sistema, brindes, adiantamentos, log de ocorrências e mensagens remotas.

Print da Tela Salve o print como prints/cfg_gerais2.png para exibir aqui.
Como acessar: Abra o módulo Gestão → clique em Configurações Gerais 2. Apenas usuários com permissão de administrador conseguem alterar esses campos.
Layout e Interface
Campo Tipo Descrição / Orientação de Suporte
Layout das Telas do Sistema
 Seleção Define o tema visual das telas do sistema. 1-Novo Layout usa a interface mais recente e é o padrão recomendado. Opções legadas mantêm o visual antigo — alterar somente se o cliente tiver dificuldade de adaptação à nova interface.
Alterar Registro do Windows p/Padrão da Automação Comercial
 S/N Quando S-Sim, ajusta entradas no registro do Windows para garantir compatibilidade com o ambiente de automação comercial (impressoras fiscais, balanças, etc.). Usar com cautela em máquinas com políticas de segurança corporativa restritivas.
Brindes e Adiantamentos
Campo Tipo Descrição / Orientação de Suporte
Permitir Brinde no Conta Assinada
 Seleção Controla se brindes são permitidos em vendas de conta assinada e como o valor é tratado. 2-Sim, somando c/valor do brinde das vendas inclui o valor do brinde no total da conta. Verificar com o cliente a política comercial antes de alterar.
Permitir Brinde no Adiantamento
 S/N Quando S-Sim, habilita o uso de brindes em operações de adiantamento. Normalmente mantido como N-Não para evitar inconsistências no fluxo de caixa e facilitar o fechamento.
Bloquear Adiantamentos em Sangrias e Suprimentos
 S/N Quando S-Sim, impede que operações de adiantamento sejam realizadas em conjunto com sangrias e suprimentos de caixa, evitando que o saldo fique inconsistente durante o processo.
Conta Assinada e Convênios
Campo Tipo Descrição / Orientação de Suporte
Permitir Cancelamento de Registros de Conta Assinada
 S/N Quando S-Sim, permite que operadores autorizados cancelem registros de conta assinada já gravados. Manter N-Não para maior controle e rastreabilidade — cancelamentos só devem ser feitos via supervisor.
Utilizar Limite Diário por Convênio p/Conta Assinada de Clientes
 S/N Quando S-Sim, aplica um teto de consumo diário por convênio na conta assinada. Útil para estabelecimentos que atendem empresas e precisam controlar o gasto diário por funcionário ou departamento.
Operação e Sistema
Campo Tipo Descrição / Orientação de Suporte
Utilizar "Log de Ocorrências"
 Seleção Define se o sistema registra um log detalhado de ocorrências operacionais. Não desativa o log. Ativar somente quando houver necessidade de rastrear eventos internos para diagnóstico de problemas — o log pode crescer e impactar performance.
Atribuir Path Temporário em Operações (Ex.:Backup)
 S/N Quando S-Sim, o sistema usa um diretório temporário para operações como backup, evitando sobrecarga ou conflito na pasta principal de instalação. Recomendado manter ativo em ambientes com múltiplos usuários.
Utilizar Rotas p/os Bairros
 S/N Quando S-Sim, habilita o cadastro e uso de rotas de entrega segmentadas por bairro. Ativar apenas para estabelecimentos que operam com delivery próprio e precisam organizar a distribuição por região.
Exibir Mensagens Remotas do Fornecedor do Sistema
 Seleção Controla se mensagens enviadas remotamente pelo fornecedor (Squalus) são exibidas ao usuário. S-Sim (todas) exibe todas as comunicações recebidas. Pode ser restringido ou desativado se o cliente não quiser interrupções durante o expediente.
Gestão › Configurações › Programa de Fidelidade

Programa de Fidelidade

Configurações de pontuação, trocas, intervalos e bloqueios do programa de fidelidade.

Print da Tela Salve o print como prints/prog_fidelidade.png para exibir aqui.
Como acessar: Abra o módulo Gestão → Configurações → Programa de Fidelidade. A tela possui dois botões no topo: Definir Pontos dos Produtos e Definir Pontos de Troca dos Produtos — ambos abrem sub-telas para configuração por produto.
Configuração Base
CampoTipoDescrição / Orientação de Suporte
Utilizar Programa de Fidelidade
 Seleção Ativa ou desativa o programa de fidelidade. Quando Não, todos os demais campos desta tela são ignorados pelo sistema. Ativar somente após configurar os demais parâmetros para evitar comportamento inesperado no PDV.
Valor
 Numérico Valor mínimo de compra (em R$) para que o cliente acumule pontos. Se 0,00, qualquer venda gera pontos. Ajustar conforme a política comercial do cliente.
Nº Pontos
 Numérico Quantidade de pontos concedidos por unidade do valor configurado. Ex.: 1 ponto a cada R$ 1,00 gasto. Funciona em conjunto com o campo Valor.
Usar Fração no Calc do Nº de Pontos
 Seleção Define se frações de pontos são consideradas no cálculo. 0-Não utilizar fração arredonda sempre para baixo, garantindo que o cliente só receba pontos inteiros. Ativar fração quando o cliente precisar de maior precisão no acúmulo.
Na troca, cada ponto equivale a quantos Reais (R$)
 Numérico Taxa de conversão de pontos em reais no momento do resgate. Ex.: R$ 0,01 significa que 100 pontos valem R$ 1,00. Conferir junto ao cliente antes de alterar para não prejudicar clientes com pontos já acumulados.
Produtos e Pontuação
CampoTipoDescrição / Orientação de Suporte
Poder Definir Produtos do Progr.Fidelidade?
 S/N Quando S-Sim, permite restringir o programa de fidelidade a um conjunto específico de produtos (configurado pelo botão Definir Pontos dos Produtos no topo da tela). Útil quando nem todos os itens do cardápio participam do programa.
Calcular Pontos de Troca por Produtos
 S/N Quando S-Sim, o custo em pontos de cada produto na troca é definido individualmente (via botão Definir Pontos de Troca dos Produtos). Quando N-Não, usa a taxa global configurada no campo Na troca, cada ponto equivale a....
Nº de Pontos para Aviso de Brinde Disponível
 Numérico Quantidade de pontos a partir da qual o sistema exibe um aviso no PDV informando que o cliente tem brinde disponível. 0 desativa o aviso.
NºPontos Padrão ao Inserir Produtos Quando Fidelidade por Produtos
 Seleção Pontuação padrão atribuída automaticamente a novos produtos cadastrados quando o programa é configurado por produto. Evita que novos itens fiquem sem pontuação definida.
Clientes e Intervalo de Acúmulo
CampoTipoDescrição / Orientação de Suporte
Valor Padrão Para Novos Clientes
 Seleção Define se novos clientes cadastrados já entram automaticamente no programa de fidelidade. Campo Programa de Fidelidade = "Sim" insere todos os novos clientes no programa por padrão.
Considerar "Qtd=1" quando venda de produtos por "kg" for menor do que 1
 S/N Quando S-Sim, vendas fracionadas (ex.: 0,3 kg) são tratadas como 1 unidade para fins de pontuação. Ativar quando o cliente quiser garantir pontos mesmo em compras de pequenas quantidades em peso.
Usar Intervalo de Data p/o Programa de Fidelidade
 S/N Quando S-Sim, o acúmulo e a troca de pontos ficam restritos ao período definido nos campos de data abaixo. Útil para campanhas sazonais ou promoções com prazo determinado.
Data Inicial p/Acúmulo de Pontos
 Data Data de início do período em que o cliente pode acumular pontos. Só tem efeito quando Usar Intervalo de Data está ativo.
Data Final p/Acúmulo
 Data Data de encerramento do acúmulo de pontos. Após essa data, vendas não geram novos pontos. Só tem efeito quando Usar Intervalo de Data está ativo.
Data Final p/Troca de Pontos
 Data Data limite para o cliente resgatar os pontos acumulados. Pode ser diferente da data final de acúmulo — ex.: acúmulo até 31/12, troca até 31/01. Só tem efeito quando Usar Intervalo de Data está ativo.
Troca de Pontos
CampoTipoDescrição / Orientação de Suporte
Permitir Inserção Manual de Pontos no Progr.Fidelidade
 S/N Quando S-Sim, operadores autorizados podem adicionar pontos manualmente ao saldo do cliente, sem vínculo com uma venda. Manter N-Não como padrão para garantir rastreabilidade — ativar somente mediante solicitação justificada do cliente.
Cálculo Automático de Pontos na Troca dos Pontos
 S/N Quando S-Sim, o sistema calcula e aplica automaticamente o desconto em reais ao efetuar uma troca de pontos, sem que o operador precise informar o valor manualmente.
Mínimo de Pontos Cliente Precisa ter para Troca
 Numérico Quantidade mínima de pontos que o cliente deve possuir para poder realizar uma troca. 0 desativa o limite mínimo. Usar para evitar resgates de valores muito baixos que gerem mais custo operacional do que benefício.
Máximo de Pontos Cliente Pode Usar na Troca
 Numérico Limite máximo de pontos que o cliente pode utilizar em uma única troca. 0 significa sem limite. Usar quando o cliente quiser controlar o desconto máximo concedido por transação.
Bloqueios de Acúmulo
CampoTipoDescrição / Orientação de Suporte
Bloquear Acúmulo de Pontos no Fidelidade nas Entregas
 S/N Quando S-Sim, vendas realizadas como entrega não geram pontos no programa de fidelidade. Útil quando o cliente quer restringir o programa ao consumo no local.
Bloquear Acúmulo de Pontos no Fidelidade p/Conta Assinada
 S/N Quando S-Sim, vendas lançadas em conta assinada não acumulam pontos. Recomendado quando o programa de fidelidade deve se aplicar apenas a pagamentos à vista ou imediatos.
Bloquear Acúmulo de Pontos no Fidelidade p/Vnd.Antecipada
 S/N Quando S-Sim, vendas antecipadas não geram pontos no programa de fidelidade. Manter conforme a política comercial do estabelecimento.
Gestão › Configurações › Venda Antecipada

Venda Antecipada

Configurações de modalidade, precificação por peso, saldo negativo e bloqueio de clientes.

Print da Tela Salve o print como prints/venda_antecipada.png para exibir aqui.
Como acessar: Abra o módulo Gestão → Configurações → Venda Antecipada. As alterações são gravadas ao clicar em Salvar. Clicar em Cancelar descarta qualquer mudança não salva.
Configurações de Venda Antecipada
CampoTipoDescrição / Orientação de Suporte
Utilizar Venda Antecipada
 Seleção Define se e como a venda antecipada é utilizada. Sim, por preço habilita o módulo com base em valor monetário por kg. Quando desativado, os demais campos desta tela são ignorados. Confirmar com o cliente a modalidade antes de alterar.
Preço do Kilo na Venda Antecipada
 Numérico Valor padrão cobrado por kg nas vendas antecipadas. Usado quando o cliente compra acima do limite definido no campo abaixo. Verificar se o preço está atualizado conforme a tabela vigente do cliente.
Se o cliente comprar menos de (Kgs)
 Numérico Quantidade mínima em kg para aplicar o preço padrão do kilo. Compras abaixo desse valor terão o preço alternativo definido no campo O preço do Kg. será então. Permite diferenciar o preço para compras pequenas.
O preço do Kg. será então
 Numérico Preço por kg aplicado quando a quantidade comprada for inferior ao limite definido no campo acima. Normalmente igual ou superior ao preço padrão, penalizando compras de menor volume.
Usar Saldo Negativo Máximo na Venda Antecipada
 S/N Quando S-Sim, o sistema permite que o cliente fique com saldo negativo na venda antecipada, até o limite definido no campo abaixo. Manter N-Não como padrão para evitar exposição de crédito não controlada.
Valor Máximo do Saldo Negativo
 Numérico Limite máximo em reais que o cliente pode ficar negativo na venda antecipada. Só tem efeito quando Usar Saldo Negativo Máximo está ativo. 0 sem limite definido.
Valor Padrão para "Bloq.p/Vnd.Antecip." no Cadastro de Clientes
 Seleção Define o status padrão do campo de bloqueio de venda antecipada para clientes novos. Não bloqueado permite que todos os novos clientes utilizem a venda antecipada por padrão. Alterar para bloquear automaticamente caso o estabelecimento prefira liberar individualmente.
Gestão › Configurações › Fechamento de Caixa

Fechamento de Caixa

Define o que é impresso e exibido no relatório de fechamento do caixa.

Print da Tela Salve o print como prints/fech_caixa.png para exibir aqui.
Como acessar: Abra o módulo Gestão → Configurações → Fechamento de Caixa. Alterações afetam o layout do relatório impresso ao fechar o caixa no PDV.
Formas de Pagamento
CampoTipoDescrição / Orientação de Suporte
Informar Valores de Formas de Pagamento
 S/N Quando S-Sim, o operador precisa informar manualmente os valores de cada forma de pagamento ao fechar o caixa (conferência de gaveta). Manter N-Não quando o estabelecimento não realiza essa conferência no fechamento.
Detalhamentos no Fechamento
CampoTipoDescrição / Orientação de Suporte
Imprimir Detalhamento dos Brindes no Fechamento do Caixa
 S/N Quando S-Sim, inclui no relatório de fechamento a lista detalhada de brindes concedidos no período. Desativar se o cliente não usa brindes ou não quer esse detalhe no relatório.
Imprimir Detalhamento dos Crediários no Fechamento do Caixa
 S/N Quando S-Sim, lista as vendas a crediário no fechamento do caixa, facilitando o controle de contas a receber abertas no período.
Imprimir Detalhamento dos Fiados no Fechamento do Caixa
 S/N Quando S-Sim, exibe no fechamento a relação de vendas lançadas como fiado. Útil para estabelecimentos que controlam fiado manualmente ao final do turno.
Separar Fiados por Vendas no Fechamento do Caixa
